# 小红书搜索Skill 🔍

通过MCP接口调用小红书搜索能力，支持关键词搜索、帖子内容读取、评论区分析，自动生成结构化总结报告。

## 触发场景
当用户指示：
- "小红书搜索 [关键词]"
- "小红书调研 [主题]"
- "小红书总结 [内容]"

时自动触发本Skill。

## 工作流

### Step 1: 执行搜索
调用xiaohongshu-mcp的`search_feeds`接口获取搜索结果：
```bash
mcporter call Xiaohongshu.search_feeds keyword="[搜索关键词]" filters='{}'
```
默认返回前20条搜索结果。

### Step 2: 筛选高价值帖子
按点赞量排序（若用户明确指示其他排序方式，按用户指示行动），选取点赞量前10的帖子作为分析样本。

### Step 3: 读取帖子详情与评论
对每篇选中的帖子，调用`get_feed_detail`接口获取完整内容和评论区：
```bash
mcporter call Xiaohongshu.get_feed_detail feed_id="[feed_id]" xsec_token="[xsec_token]" load_all_comments=true limit=20 click_more_replies=true
```
- 加载全部评论（最多20条一级评论）
- 展开二级回复
- 完整获取帖子正文、图片描述、互动数据
- 调用social-media-ad-detector skill判断该帖子是否广告，如果帖子是广告的可能性很高，降低这篇帖子在总结报告中的置信度；如果你搜索到的大部份帖子都高可能性是广告，扩大搜索范围，看更多的帖子

### Step 4: 生成结构化报告
按照以下栏目输出总结：

#### 一、整体总结
- 说明找到的帖子和阅读的帖子的数量
- 核心观点和共识
- 热门内容分类
- 普遍存在的问题或争议
- 所有事实性内容标注引用来源序号（此处序号必须与“三、参考帖子列表”中的序号一致）

#### 二、评论区分析
- 意见倾向分布（正面/负面/中性占比）
- 高频讨论点
- 典型用户评价
- 争议点分析

#### 三、参考帖子列表
| 序号 | 帖子标题 | 作者 | 发表时间 | 点赞/收藏/评论数 | 笔记ID |
|------|----------|------|----------|------------------|--------|
| 1 | [标题] | [作者名] | [YYYY-MM-DD] | 👍X/⭐Y/💬Z | [feed_id] |

## 快速模式

当用户提到使用“快速模式/快速搜索”时（例如“小红书快速搜索：高达”），执行该工作流，而不是默认工作流。

1. 执行搜索
2. 阅读前5个帖子，无需调用“获取Feed详情”接口，不阅读评论
3. 简单总结帖子内容并输出

**注意**：仅当用户明确指示“快速”搜索，才调用该工作流，否则使用默认工作流





## 出错处理

出现以下错误时，跳过此次评论访问，继续执行后续访问

```
获取总评论计数失败: All attempts fail:
#1: context deadline exceeded
#2: context deadline exceeded
#3: context deadline exceeded 
```
